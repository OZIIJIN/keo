## 프로젝트

KEO는 과거 기록과 현재 상태를 연결해
사용자의 자기이해를 돕는 앱이다.

단순 메모 앱이 아니라
기록, 연결, 패턴 발견을 중심으로 한 자기이해 앱이다.

핵심 축
- 회고 엔진
- 실행 엔진

## 저장소 구조

- `apps/backend`: 메인 백엔드
- `apps/mobile`: Flutter 앱
- `poc/`: 실험용 디렉토리

현재 명시적 요청이 없으면 `poc/recall` 작업을 우선한다.

## 개발 원칙

- 빠른 실험 우선
- 단순한 구조 유지
- 읽기 쉬운 코드 선호
- 로컬 실행 가능해야 함
- 과한 추상화 금지
- 운영용 설계 조기 도입 금지

## 작업 방식

1. 현재 작업 디렉토리의 문서 확인
2. 현재 파일 구조 확인
3. 가장 작은 동작 가능한 구현부터 진행
4. 범위를 넓히기보다 현재 목표를 먼저 만족
