## 목표

현재 POC의 1차 목표는
유사 메모 retrieval 품질 검증이다.

핵심 질문: 현재 메모가 들어왔을 때
의미상 유사한 과거 메모를 제대로 찾을 수 있는가?

## 현재 범위

포함
- 현재 메모 입력
- 테스트용 과거 메모 저장
- 임베딩 생성
- top-k 유사 메모 검색
- 검색 결과 수동 확인

제외
- 회고 문장 품질 최적화
- 패턴 분석 고도화
- 계획 기능 연결
- 인증/결제/운영 기능

## 현재 목표 흐름

현재 메모 입력
-> 임베딩 생성
-> 과거 메모 top-k 검색
-> 검색 결과 검토

## 현재 고정 기술 조건

현재 단계에서 반드시 유지할 조건:
- Embedding model: nomic-embed-text

그 외 구현 방식은 현재 POC 목적에 맞게 가장 단순한 형태를 우선한다.

## 좋은 retrieval 기준

- 키워드만 겹치는 것이 아니라 의미가 유사함
- 비슷한 감정, 문제, 맥락이 잡힘
- 엉뚱한 결과가 적음
- 사람이 봤을 때 "과거의 비슷한 생각"이라고 느껴짐

## 평가 질문

1. 현재 메모와 주제가 비슷한가?
2. 감정이나 문제 상황이 비슷한가?
3. 단순 단어 중복만으로 잡힌 것은 아닌가?
4. top-k 안에 실제로 쓸 만한 결과가 있는가?

## 완료 기준

1. 현재 메모 입력 가능
2. 임베딩 생성 가능
3. 과거 메모 검색 가능
4. top-k 결과를 사람이 검토 가능
5. retrieval 품질을 반복 개선할 수 있음

## 구현 원칙

- 가장 작은 흐름부터 구현
- retrieval을 generation과 분리해서 확인 가능하게 만들기
- 결과를 사람이 바로 읽을 수 있게 출력
- 실험용 데이터와 결과를 파일로 남기기 쉽게 만들기
